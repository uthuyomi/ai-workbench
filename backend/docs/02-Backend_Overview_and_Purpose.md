Backend Overview and Purpose

本文書の位置づけ

本書は ai-workbench Backend が「なぜ存在するのか」「何を目的としているのか」を明文化するための文書である。

本 Backend は単なる API 群や LLM ラッパーではない。 設計思想そのものが成果物であり、実装はその具現化に過ぎない。

本書は、以降のすべての仕様・実装・改修に対して 方向性を固定する役割 を持つ。


---

解決しようとしている問題

近年の AI コーディングツールは、以下の問題を内包している。

AI がコードを直接生成・編集することで、変更理由が不透明になる

判断過程がブラックボックス化し、レビューが困難になる

文脈を誤った変更が、静かにコードベースを破壊する

人間の設計判断が後追いになり、主導権を失う


これらは「AI が賢いかどうか」とは無関係に発生する構造的問題である。

ai-workbench Backend は、この構造そのものを是正するために設計されている。


---

Backend の目的

本 Backend の目的は、以下の一点に集約される。

> AI を「自動化装置」ではなく「判断補助装置」として安全に運用すること



そのために、Backend は次の役割を担う。

人間の指示を構造化し、解釈の揺れを減らす

コード変更を Diff として可視化する

変更の適用を人間の承認プロセスに委ねる

設計文書を一次情報として扱う


Backend は「速さ」や「派手さ」を追求しない。 可逆性・説明可能性・再現性 を最優先とする。


---

人間と AI の役割分担

本 Backend における役割分担は明確に定義されている。

人間の役割

要件を決定する

変更を承認または拒否する

設計方針を更新する


AI の役割

情報を整理する

変更案を生成する

判断材料を提示する


AI は決定を行わない。 決定に見える振る舞いは、設計上の欠陥とみなす。


---

Backend が目指す状態

本 Backend は、以下の状態を目指す。

変更理由を第三者に説明できる

半年後の自分が読んで理解できる

AI に改修を依頼しても設計が壊れない


Backend は「一時的に便利な道具」ではなく、 長期的に信頼できる開発基盤であることを目標とする。


---

他システムとの差別化

ai-workbench Backend は、以下の点で一般的な AI コーディングツールと異なる。

ファイルを直接編集しない

自律的にタスクを進行しない

設計文書を軽視しない


これらは制約ではなく、意図的な選択である。


---

本文書の拘束力

本文書は Backend の目的と方向性を固定する

本文書と矛盾する実装は、機能的に動作していても不正とみなす

目的の変更が発生した場合は、実装より先に本文書を更新する
